概述

计算机网络在信息时代中的作用

21世纪的重要特征是数字化，网络化，信息化，是一个以网络为核心的信息时代。

其中有三类网络为人们熟知，分别是电信网络(电话，电报，传真），有线电视网络(电视节目)，计算机网络(计算机之间传送数据)。

Internet中文译名：因特网或互联网。

互联网具有两个重要特点，分别是连通性和共享，所谓连通性，就是用户不管间隔多远，都能十分便捷地交换各种信息，就好像彼此连通在一起一样。共享指的是资源共享，包括软件共享，信息共享，甚至可以是硬件共享。

互联网

计算机网络由若干个节点和连接这些节点的链路构成，多个网络通过路由器连接起来，构成互连网，也称网络中的网络。

网络把许多计算机连接起来，而互连网把许多网络连接起来，连接在网络上的计算机称为主机。

互联网的组成

  1. 边缘部分：由所有连接在互联网上的主机组成，用户直接使用。
  2. 核心部分：由大量网络和连接这些网络的路由器组成，为边缘部分提供服务(提供连通性和交换)。

处在互联网边缘部分的就是连接在互联网上的所有主机，称为端系统。

计算机间的通信指的是主机A的某个进程与主机B的某个进程的通信，通信方式通常可分为两大类：

  1. 客户-服务器方式：

    客户是服务请求方，被用户调用后，向远方服务器发起请求，需要知道服务器程序的地址。

    服务器是服务提供方，可同时处理多个客户的请求。

  2. 对等连接方式(P2P):

    两台主机通信时，并不区分客户方和服务方，进行对等的通信。

c++模板类无法访问继承的子类的成员

这几天在使用Visual Studio 2019的时候遇到了一个问题，创建了一个继承了类模板的类模板，但是其成员函数无法访问父类成员，具体问题如下：

#include <iostream>
using namespace std;

template<class T>
class A {
public:
    T data;
    T test(){};
};

template<class T>
class B:public A<T> {
public:
    void test01() { data; }
};



int main() {
    B<int> a;
    return 0;
}

VS 2019并不会报语法错误，当编译时会报未找到data标识符。

其原因如下：

根据c++标准，对模板代码中的名字的查找，分为两个部分：

  • 模板定义阶段：刚被定义时，只有模板中独立的名字(可以理解为和模板参数无关的名字)参与查找。
  • 模板实例化阶段：实例化代码时，非独立的名字才参加查找。

对于上述例子中，在模板定义阶段，B的基类A<T>依赖于模板参数T，所以对于B来说，A<T>并不存在，于是A<T>::f()也不存在，但此时这段代码是合法的，因为编译器将f认为是一个非成员函数。当进入实例化阶段时，由于之前已将f()认为时非成员函数，这个阶段编译器会从非成员函数里查找f()，此时自然找不到f()

解决方法

只要告诉编译器f()时成员函数即可，因此有以下两种方法：

  1. 添加this，告诉编译器f()时成员函数，即this -> f()。
  2. 直接声明f()是A<T>的成员函数，如A<T>::f()。

类模板分文件编写出现的问题

将类模板的声明与定义写在不同文件中的时候，在预编译阶段并没有报错，而在实例化模板后，会报链接器错误。

相关例子如下：

//在test.h文件中
#pragma once

template<class T>
class Test {
public:
	void A();
};
#include "test.h"

template<class T>
void Test<T>::A() {};

在语法上并没有错误，但一旦实例化后编译器将会提示链接器错误。

#include <iostream>
#include "test.h"

using namespace std;

int main() {
	Test<int> test;
}

出现该问题的原因

  1. 首先我们需要了解c++分文件编写的原理，我们一般在.h文件里声明需要使用的类和函数，在.cpp文件里对函数做出定义，在.cpp文件里需要添加相应头文件。实际上，在编译阶段，编译器会将.h文件中的内容拷贝到.cpp文件中，然后在cpp文件中找到相应函数声明与定义。
  2. 我们也需要了解类模板的原理，类模板并不是一个类，而是一种模板，其编译和声明是在编译阶段才能确定的。

线程简介

  1. 进程

    狭义理解就是操作系统中一段程序的执行过程。那么广义上，进程是指一个具有一定独立功能的程序操作系统中关于某个数据集合进行的一次运行活动。是操作系统程序动态执行的基本单元。在传统的操作系统中，进程既是一个操作系统的基本分配单元，也是操作系统的基本执行单元。

  2. 线程

    线程是进程中执行运算的最小单元，是操作系统执行处理机制的基本单位。每个进程至少有一个线程，线程可以利用进程所拥有的资源执行调度和运算。

  3. 多线程

    在一段完整的代码中，往往会有需要独立的代码模块，而这些独立运行的程序片段叫作线程,利用多个线程编程的概念就叫作多线程处理(多线程编程)，多线程是为了同步完成多项任务，不是为了提高运行效率，而是为了提高资源使用效率来提高系统的效率。多线程是在程序在同一时间需要完成多项任务的时候实现的。多线程的目的仅仅是为了提高资源利用效率。各个线程执行自己的任务，这些线程可以”同时进行“。同时进行并非同一时刻进行，而是在某一时间段内，完成所有任务，任务的运行有先后顺序。

核心概念

  • 线程就是独立的执行路径。
  • 在程序运行时，即使没有自己创建线程，后台也会有多个线程，如主程序，gc线程。
  • main()称为主线程，为系统的入口，用于执行整个程序。
  • 在一个线程中，如果开辟了多个线程，线程的运行由调度器安排调度，调度器是与操作系统紧密相关的，先后顺序是不能人为的干预的。
  • 对同一份资源操作时，会存在资源抢夺的问题，需要加入并发控制。
  • 线程会带来额外的开销，如cpu调度时间，并发控制开销。
  • 每个线程在自己的工作内存交互，内存控制不当会造成数据不一致。

观察线程状态

  • Thread.State

    线程可以处于一下状态

    • NEW

      尚未启动的线程处于此状态

    • RUNNABLE

      在java虚拟机中执行的线程处于此状态

    • BLOCKED

      被阻塞等待监视器锁定的线程处于此状态

    • WAITING

      正在等待另一个线程执行特定动作的线程处于此状态

    • TIMED_WAITING

      正在等待另一个线程执行动作达到指定等待时间的线程处于此状态

    • TERMINATED

      已退出的线程处于此状态

      实例

      public class testState {

        Thread t = new Thread(()->{
            for(int i = 0; i < 5; i++) {
                try {
                    Thread.sleep(1000);
                }catch(InterruptedException e) {
                    e.printStackTrace();
                }
            }
            System.out.println("This is the end!");

        //观察状态
        Thread.State state = t.getState();
        System.out.println(state);

        //观察启动后
        t.start();
        state = t.getState();
        System.out.println(state);

            Thread.sleep(100);
            state = t.getState();
            System.out.println(state);
        }
    }
}

线程同步 🌟

  • 处理多线程问题时，多个线程访问同一个对象，并且某些线程还想修改这个对象，这时候我们就需要线程同步。线程同步其实就是一种等待机制，多个需要同时访问此对象的线程进入这个对象的等待池形成队列，等待前面线程使用完毕，下一个线程在使用。

  • 为了保证数据在方法中被访问时的正确性，在访问时加入锁机制synchronized,当一个线程获得对象的排它锁，独占资源，其他线程必须等待，使用后释放锁即可。存在以下问题：

    • 一个线程持有锁会导致其他所有需要次锁的线程挂起。
    • 在多线程竞争下，加锁，释放锁会导致比较多的上下文切换和调度延时，引起性能问题。
    • 如果一个优先级高的线程等待一个优先级低的线程释放锁，会导致优先级倒置，引起性能问题。

  • 由于我们可以通过private关键字来保证数据对象只能被方法访问，所以我们只需要针对方法提出一套机制，这套机制就是synchronized关键字，它包括两种用法：synchronized方法和synchronized块

  • synchronized方法控制对“对象"的访问，每个对象对应一把锁，每个synchronized方法都必须获得调用该方法的对象的锁才能执行，否则线程会阻塞，方法一旦执行，就独占该锁，直到该方法返回才释放锁，后面被阻塞的线程才能获得这个锁，继续执行

    //同步方法 public synchronized void method(int args) {}

    缺陷：若将一个大的方法申明为synchronized将会影响效率

同步块

  • 同步块：synchronized(Obj) {}
  • Obj称之为同步监视器
    • Obj可以是任何对象，但是推荐使用共享资源作为同步监视器
    • 同步方法中无需指定同步监视器，因为同步方法的同步监视器就是this这个对象本身，或者是class
  • 同步监视器的执行过程
    1. 第一个线程访问，锁定同步监视器，执行其中代码。
    2. 第二个线程访问，发现同步监视器被锁定，无法访问。
    3. 第一个线程访问完毕，解锁同步监视器。
    4. 第二个线程访问，发现同步监视器没有锁，然后锁定并访问。

最长递增子序列个数

发表于 2021-11-22 |
字数统计: 1.2k | 阅读时长 ≈ 5
  1. 最长递增子序列个数

难度：⭐⭐⭐

原题

给定一个未排序的整数数组，找到最长递增子序列的个数。

示例：

输入: [1,3,5,4,7]
输出: 2
解释: 有两个最长递增子序列，分别是 [1, 3, 4, 7] 和[1, 3, 5, 7]。

注意: 给定的数组长度不超过 2000 并且结果一定是32位有符号整数。

解题思路：

  1. 通过动态规划求解

    • 可以把问题拆分成两部分，先求出最长递增子序列，再求个数。

      使用d[i]数组来记录到第i个元素时最长递增子序列的长度。

      其中，d[i]的状态方程为
      $$
      d[i] = max(d[j]) + 1\quad j < i
      $$

      可以使用两个for循环来进行状态压缩，从而求出d[i]。

    • 可以使用max[i]数组来记录以第i个元素结尾的最长递增子序列的个数。

      在上一步求d[i]时，可以更新max[i]的值。

      在nums[i] > nums[j] 时：

      当 d[j] + 1 > d[i] 时，说明找到到第i个元素的更长的递增子序列，此时令

      d[i] = d[j] + 1;
max[i] = max[j];

      当 d[j] + 1 == d[i] 时。

      max[i] += max[j];

      因为max[i]表示的是到以第i个元素结尾的最长递增子序列个数，而nums[i] > nums[j], 说明以第i个元素结尾的递增子序列又多了一个分支，即第j个元素。

    • 用ans存储答案，maxLen记录当前最长子序列的长度。

      在求d[i]的值时，可以顺便进行比较，将ans不断更新。

    源代码

    class Solution {
public:
    int findNumberOfLIS(vector<int>& nums) {
        int dp[2001], max[2001];
        int maxLen = 0;
        int num = nums.size();
        int ans = 0;
        for(int i = 0; i < num; ++i)
        {
           dp[i] = 1;
           max[i] = 1;
           for(int j = 0; j < i; ++j)
           {
               if(nums[i] > nums[j])
               {
                   if(dp[j] + 1 > dp[i])
                   {
                       dp[i] = dp[j] + 1;
                       max[i] = max[j];
                   }
                   else if(dp[j] + 1 == dp[i])
                   {
                       max[i] += max[j];
                   }
               }
           }
           if(dp[i] > maxLen)
           {
               maxLen = dp[i];
               ans = max[i];
           }
           else if(dp[i] == maxLen)
           {
               ans += max[i];
           }
        }
        return ans;
    }
};

  2. 贪心+前缀和+二分查找

    • 同样先将问题看成两部分，第一步先求出最长子序列。可以使用一个$dp[i]$数组，$dp[i]$表示的是长度为 i 的递增子序列中末尾最小的。

      如：
	1 3 5 4 7
其中长度为3的递增子序列有：
	1 3 5
	1 3 4
	1 3 7
	3 5 7
此时dp[3] = 4

      遍历nums数组，每次从dp数组中找出满足$dp[i] < nums[j] < dp[i + 1]$的 i, 然后更新$dp[i + 1]$的值，保证$dp[i+1]$是长度为 i + 1的递增子序列中末尾值最小的。遍历完nums数组之后，只需看dp数组的长度即可求得最长递增子序列。但为了求最长递增子序列的个数，我们需要将更新的历史信息记录下来，使用二维数组$dp[i][j]$来存储，$dp[i][j]$表示的是第j个以长度为i的递增子序列的末尾的值。

      如：
	1 3 5 4 7
其中长度为2的递增子序列有：
	1 3 5
	1 3 4
	1 3 7
	3 5 7
此时
	dp[1][0] = 1
	dp[2][0] = 3
	dp[3][0] = 5
	dp[3][1] = 4
	dp[4][0] = 7

      另一个例子

      来自leecode题解区

    • 使用cnt数组来储存最长递增子序列个数。其中$cnt[i][j]$记录了以$dp[i][j]$结尾的最长上升子序列个数，为计算$cnt[i][j]$，我们可以将所有满足$dp[i-1][k]<d[i][j]$的$cnt[i-1][k]$累加到$cnt[i][j]$，这样最终答案就是$cnt[maxLen]$的所有元素之和。

    • 可以使用前缀和进行优化，让每个$cnt[i][0] = 0$，此时$d[i][j]$对应的最长上述子序列就是$cnt[i-1][-1] - cnt[i-1][k]$，这里的-1表示数组的最后一个元素。由于存进数组中的数都是有序的，可以使用二分查找加快查找效率。

    代码

    class Solution {
public:
    int findNumberOfLIS(vector<int>& nums) {
        vector<vector<int>> dp, cnt;
        for(int v : nums)
        {
            int min = 0;
            int max = dp.size();
            int i, c = 1;
            while(min < max)
            {
                i = (max + min) / 2;
                if(v > dp[i].back())
                    min = i + 1;
                else
                    max = i;
            }
            i = min;
            if(i > 0)
            {
                int k;
                min = 0;
                max = dp[i - 1].size();
                while(min < max)
                {
                    k = (max + min) / 2;
                    if(dp[i - 1][k] < v)
                        max = k;
                    else
                        min = k + 1;
                }
                k = min;
                c = cnt[i - 1].back() - cnt[i - 1][k];
            }
            if(i == dp.size())
            {
                dp.push_back({v});
                cnt.push_back({0, c});
            }
            else
            {
                dp[i].push_back(v);
                cnt[i].push_back(cnt[i].back() + c);
            }
        }
        return cnt.back().back();
    }
};

难度：⭐

原题：

给定一个二叉树的根节点root, 返回它的中序遍历。

示例：

输入：root = [1, null, 2, 3]
输出：[1, 2, 3]

解题思路：

中序遍历即先访问左节点，再访问根结点，最后访问右节点。

  • 通过递归算法求出中序遍历系列。

    class Solution {
public:
    void inorder(TreeNode* root, vector<int>& res) {
        if (!root) {
            return;
        }
        inorder(root->left, res);
        res.push_back(root->val);
        inorder(root->right, res);
    }
    vector<int> inorderTraversal(TreeNode* root) {
        vector<int> res;
        inorder(root, res);
        return res;
    }
};

  • 通过迭代算法求出中序遍历

    1. 从根结点开始，访问左节点。每次访问都将左节点进栈。(注意：一开始的根节点也需要进栈)
    2. 取出栈顶元素并访问该元素。让current指向该元素的右节点。

    **注意：**结束循环的条件。

    源代码：

    /**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left), right(right) {}
 * };
 */
class Solution {
public:
    vector<int> inorderTraversal(TreeNode* root) {
        TreeNode* current = root;
        vector<int> ans;
        stack<TreeNode*> b;
        do{
            while(current!=nullptr)
                {
                    b.push(current);
                    current = current->left;
                }
            if(!b.empty())
            {
                current = b.top();
                b.pop();
                ans.push_back(current->val);
                current = current->right;
            }
        }while(!b.empty()||current!=nullptr);
        return ans;
    }
};

题意理解
有n块多米诺骨牌，每张牌的两面分别是0~6的数字，我们可以重新排列这些骨牌的顺序，并决定是否翻转这些骨牌(翻转用’-'表示)，使得每个骨牌后面的数字与相邻的骨牌前面的数字相同。输出一种可行的方案。
样例： 4 2 2 1 1 2 2 4 4 6

看到题目之后，首先想到判断是否为半欧拉图或欧拉图

  • 半欧拉图：在一个图中，如果存在一条通过图中每条边一次且仅一次行遍图中每个顶点的通路且不存在通过图中每条边一次且仅一次行遍图中每个顶点的回路，则称G是半欧拉图。
  • 欧拉图：通过图(无向图或有向图)中所有边且每边仅通过一次通路，相应的回路称为欧拉回路。具有欧拉回路的图称为欧拉图。

判断是否为半欧拉图: 当无向图G是连通的且恰有两个奇度顶点。

解题思路

  1. 首先创建一个edge结构体，将边信息记录。为方便后面查找相邻边，可添加next属性记录相邻边所在行数，在源代码中用head数组表示。
  2. 判断是否为欧拉图或半欧拉图，不是则直接输出 No solution
  3. 从某个奇度顶点开始，寻找与之相邻的一条边，当下一条边确定后，再将这条边加入ans数组中。遍历的同时，使用visit数组记录浏览过的点。
  4. 判断每个点是否都被浏览过。

JavaFX是用于构建富互联网应用程序的Java库。 使用此库编写的应用程序可以跨多个平台一致运行。使用JavaFX开发的应用程序可以在各种设备上运行，如台式计算机，手机，电视，平板电脑等。要使用Java编程语言开发GUI应用程序，程序员依赖于诸如高级窗口工具包(AWT)和Swings之类的库。在JavaFX出现之后，这些Java程序开发就可以有效地利用丰富的类库来开发GUI应用程序。

    • JavaFX应用程序使用的是舞台和场景的隐喻，因为它们就像剧院 中的舞台和场景一样。Stage类是顶层的JavaFX容器，在我们的例子中，它对应于窗口。每个Stage都有一个相关联的Scene对象，这个Scene对象包含一组称为场景图的节点。这些节点由Node类表示，描述了应用程序的一个场景，就像剧本、演员和道具描述了戏剧或电影的一个场景一样。在JavaFX中，场景图是层次化的节点集，其中根节点位于树的顶端。在根节点之下，我们可以创建包含布局构件的子树，例如可以绘制到场景中的面板、按钮、文本或形状。节点具有属性，包括文本、尺寸、位置、颜色和可视化效果等。

事件驱动的三种方式

事件是一种对象，它们表示会触发响应的某个用户动作，例如用鼠标点击按钮。当一个对象产生某个事件时，就称它触发了某个事件。例如，当用户点击按钮时，该按钮就会触发一个事件。在JavaFX应用中，每个可以触发事件的对象都可以有一个或多个会自动接收事件的监听器对象。监听器对象中包含的方法指定了在该监听器接收到各种事件时应该做些什么。这些方法称为事件处理器。 接下来，我们来讨论选择监听器类的3种选项。无论选择哪种选项，监听器类都必须实现EventHandler 。

在单独的类中处理事件

创建一个HandleButtonClick类。

import javafx.event.ActionEvent;
import javafx.event.EventHandler;

/**
This class handles a button click and outputs a message.
The handle method is invoked when the button is clicked.
*/
public class HandleButtonClick implements EventHandler<ActionEvent>
{
	private String message;
	public HandleButtonClick()
	{
		message = "It is sunny!";
	}
	public HandleButtonClick(String customMessage)
	{
		message = customMessage;
	}
	@Override
	public void handle(ActionEvent event)
	{
        System.out.println(message);
    }
}

为了将按钮点击事件与HandleButtonClick对象关联起来，我们使用了setOnAction方法。

HandleButton clickEvent = new HandleButtonClick();

在主GUI应用类中处理事件

让ButtonDemo类自身实现EventHandler 接 口 。这意味着我们需要在ButtonDemo类中添加handle方法。因为这个方法在ButtonDemo内部，所以它可以访问作为该类组成部分的任何成员变量，例如其他的UI控件。

import javafx.application.Application;
import javafx.scene.Scene;
import javafx.stage.Stage;
import javafx.scene.layout.VBox;
import javafx.scene.control.Button;
import javafx.event.ActionEvent;
import javafx.event.EventHandler;

/**
Demonstration of event handling within the ButtonDemo2 class.
*/
public class ButtonDemo2 extends Application implements EventHandler<ActionEvent>
{
   private Button btnSunny;
   private Button btnCloudy;

   public static void main(String[] args)
   {
      launch(args);
   }

   @Override
   public void handle(ActionEvent event)
   {
	   // This method can access the member variables
	   // which reference the other GUI controls
	   if (event.getSource() instanceof Button)
	   {
		  Button btnClicked = (Button) event.getSource();
       	  if (btnClicked.getText().equals("Sunny"))
       	  {
			  // Disable the cloudy button if sunny clicked
			  btnCloudy.setDisable(true);
		  }
       	  else if (btnClicked.getText().equals("Cloudy"))
       	  {
			  // Disable the sunny button if cloudy clicked
			  btnSunny.setDisable(true);
		  }
	  }
   }

   @Override
   public void start(Stage primaryStage) throws Exception
   {
	  VBox root = new VBox();
	  btnSunny = new Button("Sunny");
	  btnCloudy = new Button("Cloudy");

	  btnSunny.setOnAction(this);
	  btnCloudy.setOnAction(this);

	  root.getChildren().add(btnSunny);
	  root.getChildren().add(btnCloudy);

   	  Scene scene = new Scene(root, 300, 100);
      primaryStage.setTitle("Button Demo 2");
      primaryStage.setScene(scene);
      primaryStage.show();
   }
}

上述代码确实产生了一个新的应用。由于现在只有单个对象（当前对 象）来处理两个按钮的点击事件，所以如果我们希望区分到底点击了 哪个按钮，就必须在handle方法中进行确认。在之前的版本中，我们区分哪个按钮点击的方式是每个按钮关联不同的HandleButtonClick实例。实现按钮区分的一种方式是调用getText方法，它会获取按钮上的文本。然后, 我们检查文本的内容并作出相应的动作。

在匿名内部类中处理事件

这种技术有一个优势，就是赋予了事件处理器类访问主GUI类中所有成员变量的权限，同时又允许我们可以创建不同的监听器实例。创建实现了EventHandler接口的匿名内部类的格式如下：

new EventHandler<ActionEvent>()
{
    // 对类定义任意成员变量或方法
}

我们可以将这段代码插到setOnAction方法中，并在其中填入handle方法的实现。在这个例子中，我们的实现是将ButtonDemo类标签中的文本修改为"It is sunny！"：

btnSunny.setOnAction(new EventHandler<ActionEvent>()
{
    @Override
    public void handle(ActionEvent event)
    {
        lblMessage.setText("It is sunnt!");
    }

数据库：

DataBase,简称DB.
数据库就是按照一定格式存储数据的一些文件的组合。实际上就是一堆文件，这些文件中存储了具有特定格式的数据。

数据库管理系统：

DataBaseManagement,简称DBMS.
数据库管理系统是专门用来管理数据库中数据的，数据库管理系统可以对数据库当中的数据进行增删改查。
常见的数据库管理系统：
MySQL,Oracle,MS SqlServer,DB2,sybase等…

SQL:结构化查询语言

程序员需要学习SQL语言，编写SQL语句，然后DBMS负责执行SQL语句，最终来完成数据库中数据的增删改查操作。

mysql基本操作:

启动和关闭mysql服务:
 net start mysql; 启动
 net stop mysql; 关闭
退出:
 exit
登录:
 mysql -uroot -p密码

查看mysql有哪些数据库
show databases;

查看数据库中有哪些表
show tables;

查看数据库的版本号
select version

SQL语句的分类

SQL语句有很多，最好进行分类，容易记住
 DQL:Data Definition Languages(数据查询语言)
    select…
 DML:Data Manipulation Language(数据查询)
    insert delete update
 DDL:Data Data Definition Languages(数据操纵语句)
    create, drop, alter
 TCL:Transaction Control Language(事务控制语句)
    commit, rollback
  DCL:Data Control Language(数据控制语言)
    授权grant, 撤销revoke…

简单查询

查看表中所有数据:
select * from 表名

不看表名，只看结构:
desc 表名;

查询一个字段:
select 字段名 from 表名;
多个字段名中间用逗号隔开:select 字段名,字段名 from 表名

给查询的列起别名:
select 字段名 as 别名 from 表名;

条件查询:
select 字段名 from 表名 where 条件;

条件
= 等于
<>或!= 不等于
< 小于 > 大于
between … and …或>= and <= 表示两值之间
is null为null
in 表示包含where job in(‘SALE’)
like 表示模糊查询
名字以T结尾like ‘%T’
名字以K开始like ‘K%’
第三个字母是R的like ‘_R%’

排序

select 字段名 from 表名 order by 字段名;默认是升序
select 字段名 from 表名 order by 字段名 desc;表示降序（asc为升序)
