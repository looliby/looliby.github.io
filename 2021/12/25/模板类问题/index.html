c++模板类无法访问继承的子类的成员

这几天在使用Visual Studio 2019的时候遇到了一个问题，创建了一个继承了类模板的类模板，但是其成员函数无法访问父类成员，具体问题如下：

#include <iostream>
using namespace std;

template<class T>
class A {
public:
    T data;
    T test(){};
};

template<class T>
class B:public A<T> {
public:
    void test01() { data; }
};



int main() {
    B<int> a;
    return 0;
}

VS 2019并不会报语法错误，当编译时会报未找到data标识符。

其原因如下：

根据c++标准，对模板代码中的名字的查找，分为两个部分：

  • 模板定义阶段：刚被定义时，只有模板中独立的名字(可以理解为和模板参数无关的名字)参与查找。
  • 模板实例化阶段：实例化代码时，非独立的名字才参加查找。

对于上述例子中，在模板定义阶段，B的基类A<T>依赖于模板参数T，所以对于B来说，A<T>并不存在，于是A<T>::f()也不存在，但此时这段代码是合法的，因为编译器将f认为是一个非成员函数。当进入实例化阶段时，由于之前已将f()认为时非成员函数，这个阶段编译器会从非成员函数里查找f()，此时自然找不到f()

解决方法

只要告诉编译器f()时成员函数即可，因此有以下两种方法：

  1. 添加this，告诉编译器f()时成员函数，即this -> f()。
  2. 直接声明f()是A<T>的成员函数，如A<T>::f()。

类模板分文件编写出现的问题

将类模板的声明与定义写在不同文件中的时候，在预编译阶段并没有报错，而在实例化模板后，会报链接器错误。

相关例子如下：

//在test.h文件中
#pragma once

template<class T>
class Test {
public:
	void A();
};
#include "test.h"

template<class T>
void Test<T>::A() {};

在语法上并没有错误，但一旦实例化后编译器将会提示链接器错误。

#include <iostream>
#include "test.h"

using namespace std;

int main() {
	Test<int> test;
}

出现该问题的原因

  1. 首先我们需要了解c++分文件编写的原理，我们一般在.h文件里声明需要使用的类和函数，在.cpp文件里对函数做出定义，在.cpp文件里需要添加相应头文件。实际上，在编译阶段，编译器会将.h文件中的内容拷贝到.cpp文件中，然后在cpp文件中找到相应函数声明与定义。
  2. 我们也需要了解类模板的原理，类模板并不是一个类，而是一种模板，其编译和声明是在编译阶段才能确定的。
